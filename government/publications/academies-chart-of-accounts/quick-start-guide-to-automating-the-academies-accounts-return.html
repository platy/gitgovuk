Why automate?

Over the last few years, the Department for Education (DfE) has been working closely with financial management system (FMS) suppliers in the sector to enable new automation functionality within their software. The automation functionality has been used by trusts for their live AR collections and we have received positive feedback confirming the time saving potential for the sector. This guide shows how simple it is to get set up.

There are many benefits of using automation - these include: pre-population of your trial balance data within the accounts return; removing repetitive data entry for information already held in your FMS; speeding up submission time; and providing an audit trail via automated mapping document. The DfE also envisage there will be further benefits as we explore other uses for automation in sector reporting.

For more information on how automation can help in the completion of the academies accounts return and what steps you need to take to use automation, please view the video below.

Automating your Academies Accounts Return

How does automation work?

This new technology enables you to send data directly from your FMS. This flowchart provides a summary of the steps taken during automation.

The trial balance data you apply will then be used to pre-populate up to 80% of your AR.

Can I use automation?

We have been working with FMS suppliers who cover the majority of the sector. Response to the project has been positive and most suppliers have completed the necessary changes to their product.

Please contact your FMS supplier to understand if they have this automation functionality set up and how to use it to send your financial data.

Three quick steps to automating your return

Step 1: Allocate someone in your trust to transfer the data

What do I need to do?

  1. ensure the user sending the data has an IDAMS account
  2. ensure this user has been allocated the ‘data transfer approver’ role

A super-user must assign or approve all requests for the data transfer approver role, more information can be found here (IDAMS page).

Note: Trusts retain control over when their data is shared with the DfE. Data will only be shared with the DfE when a user, assigned by the trust, chooses to send their FMS data and will only be accepted into the online form submissions, when a user, designated by the trust, applies it.

Step 2: Send your data to the DfE

Within your financial management system (FMS), there will be an option to send your data to the DfE. Please contact your FMS supplier for detailed instructions on how to send your data to the DfE from your FMS. The process may vary between suppliers.

Once you have sent data from your FMS, you will be directed to IDAMS to confirm your username, password and that you are happy for your financial data to be transferred to the DfE.

Step 3: Apply the data to your accounts return

If you are already logged into the AR when you submit the data via your FMS, you will see a data uploads link with a pending tag appear in the top right hand corner of the screen which will take you to the data uploads page.

If you are not logged in, when you next sign into your AR you will be taken straight to the data uploads page where you will be able to decide what to do with your data.

This screen will present you with the following options:

  • Apply: the data will be used to pre-populate the relevant trial balance cells in your AR. Note that if there is already data in these cells it will be overwritten by this new data upload. This only applies to data that can be automated, it will not overwrite non-trial balance cells, as these can only be completed manually.

  • Reject: the data is deleted and will not pre-populate your AR. You will receive a warning message to confirm this before the rejection is performed. If you decide that you should have applied the upload, you can resubmit this data or any amended data again via your financial management system.

  • Ignore: the data will not pre-populate your AR but it is saved in the background. The data uploads link will remain in the top right-hand corner so that you can return to this screen later to accept or reject it. If you submit more than one set of data without accepting or rejecting it, only the latest submitted data will be available. The data uploads page will keep track of your data acceptance/rejection history.

What next?

Once you have applied the data to your form, you still need to follow the normal form process and check it is correct, clear any validations and add any additional information that is not held in your finance system before submitting.

How do I know what data has come from my financial management system?

To easily identify which cells have been pre-populated from your finance system (rather than completed manually) these cells have been tagged with ‘Automated’ underneath them. If you manually change the data by overtyping in the field the tag will change to ‘Modified’ and the cell outline colour will change from grey to blue as illustrated below.

Automated draft trust financial statements

This functionality will enable trusts sending automated data into the accounts return on-line form, to submit their trusts trial balance data directly from the FMS to auto-populate their draft financial statements.

The ‘Draft trust financial statements’ workbook is available to download and test this year. The primary purpose is to obtain feedback ahead of a wider release next year.

Please send any feedback you may have to API.Finance@education.gov.uk to enable us to improve this functionality.

Data upload

Trusts need to upload their FMS data using the guide to automating the academies accounts return.

Data access

For the auditors to gain access to the financial statements data, the trust will need to access the AR and select the subheading ‘External Auditor details’, found under the heading ‘Overview’, and ensure the information is correct for the auditor conducting the testing.

Data transformation process

Trusts can generate their draft financial statements by clicking on the ‘Draft Trust Financial Statements’ link, which is located in the right hand menu of the index page of the AR and is highlighted with a red box in the image below. When clicked, this link displays options for the user to download the data into an Excel or PDF file format.

Draft trust financial statements workbook

The workbook uses the trusts automated data to produce draft financial statements in the Coketown format. An ‘Instructions’ tab has been included at the beginning of the workbook to aid with understanding the workbook.

Workbook data population

The workbook can be updated with new data by uploading in a new version of the trusts FMS upload file to the AR form. It will populate this template and be ready to download. There is no need to accept the data into the AR form first.

Each new automated data submission generates a new ‘Draft trust financial statements’ workbook. Older versions will not be retained. If you wish to keep older versions these will need to be downloaded and stored locally.

Workbook navigation

The ‘Note names’ tab enables easier navigation and the user can move around the workbook in the following ways:

  • each note is hyperlinked, and the user can navigate to them easily from the ‘Notes name’ tab
  • alternatively, the user can right click on the arrows found in the bottom left hand corner of the workbook and select the appropriate tab from the list
  • links are also available to the notes from the relevant references on the primary statements
  • the user can also navigate back to the ‘Notes name’ tab from any of the notes by clicking on the note name in the top left cell (cell A1)
  • the user can navigate to the relevant tabs directly from the B/S, SoFA, and CF tabs by clicking on the note number, found in Column C ‘B/S’ tab, or column B (CF and SoFA tab)

The ‘Note names’ tab

The ‘Note names’ tab can be used to alter the note numbering dynamically by deleting any number. An example is shown below where you may wish to delete Note 7 (Expenditure).

The ‘Mapping’ tab

The ‘Mapping’ tab shows the downloaded CoA data from the trusts FMS data upload. This shows how the account codes map to the draft trust financial statements workbook and the AR.

Selecting the ‘+’ above the stated columns unhides the relevant sections for you to view.

All cells can be overwritten, but many are formula driven. If you change them, the formulas may no longer work. Where data is not available from the automated feed, you can manually type information in. We suggest adding/subtracting from existing formulas rather than overwriting where this is necessary. If you find there are errors in the mapping, please feed this back via API.Finance@education.gov.uk so we can improve the workbook.

Please note the following with regards to account codes:

  • if your trust has added its own account codes, these will be rolled up to the DfE’s account code
  • if an account code is being used which is currently not in our chart of accounts, for example, the trust has used a code added to the following version of the CoA, then it must be added manually
  • if there are any cases where an account code is missing from the current collection periods CoA, please feed this back to us via API.Finance@education.gov.uk

Prior year data

The prior year tab is currently unavailable, but the data can be added manually.

Workbook data types

The workbook will need to be reviewed and updated with any changes needed for pre-populated data and also for non-trial balance data so that the trust has a full data set for the draft financial statements.

The table below describes the action required in different tabs of the workbook to provide the trust with a full set of financial statements data.

Tab colour Trial balance type Action
Green Trial balance driven All fields in the tab will be completed by automated feed (data dependant) and manual input is only required if the user wishes to update the pre-populated data
Amber Partially trial balance driven Some fields in the tab will require manual input
Red Non-trial balance information All fields in the tab will require manual input
